Anvil is not magic because it generates code. It is useful because the CLI, MCP
server, skill, and hooks all agree on what an operation *means* — its effect,
risk, idempotency, and whether it needs confirmation — from one canonical model
(AIR). Everything downstream is a projection of that model, so the surfaces can
never quietly disagree.

## The loop

<Tabs>
  <TabItem label="Compile → inspect → approve → deploy">

```bash
# 1. Compile a spec into a full bundle (CLI + MCP + skill + hooks)
node packages/cli/dist/bin-anvil.js compile <spec> --manifest <manifest> --out <dir>

# 2. Read every operation's effect, risk, and idempotency
node packages/cli/dist/bin-anvil.js inspect <dir>

# 3. Expose an operation only after inspecting its risk
node packages/cli/dist/bin-anvil.js approve <dir> <operation-id>

# 4. Package the skill and deploy the server
node packages/cli/dist/bin-anvil.js package skill <dir>
```

Non-idempotent mutations land as `review_required` — they are not exposed until
you approve them. That refusal is the feature.

  </TabItem>
  <TabItem label="Build & test">

```bash
pnpm install
pnpm build      # compile every package
pnpm test       # the safety contract's own conformance suite
pnpm typecheck
```

  </TabItem>
